# 🌊 Aquarine Central Database & Hub

Bienvenido a la documentación oficial y base de datos centralizada de **Aquarine**. Este repositorio ha sido diseñado para actuar como el núcleo informativo, diario de desarrollo (*DevLog*) y archivo histórico de todo el ecosistema de aplicaciones y experimentos de ciberseguridad/malware educativo vinculados a este proyecto.

El propósito principal de este espacio es centralizar el flujo de trabajo, documentar el código y servir como el tablón oficial de anuncios para coordinar los avances de las dos ramificaciones principales del proyecto: la versión histórica y la línea de desarrollo actual.

---

## 🏗️ Estructura del Ecosistema Aquarine

El proyecto está dividido en dos etapas fundamentales que muestran la evolución técnica del desarrollo:

1. **Aquarine Legacy Beta Edition (Archivo Histórico):** Representa los cimientos del proyecto. Es el código original desarrollado cuando recién exploraba la plataforma de GitHub. Se mantiene intacto en su propio espacio por su inmenso valor nostálgico, como registro de progreso y para recibir parches menores de mantenimiento técnico o estabilidad estructural.
2. **Aquarine Versión Normal (Desarrollo Activo):** La línea principal orientada al análisis de virus y pruebas de concepto en entornos móviles Android. Utiliza la potencia lógica y visual de Sketchware para compilar payloads visuales, simuladores de comportamiento malicioso y herramientas de investigación educativa en formato APK.

---

## 🗃️ ¿Qué encontrarás en esta Base de Datos?

Este repositorio no contiene ejecutables dañinos; es una biblioteca de datos diseñada para:
* **Planificación del Desarrollo (Roadmap):** Registro público de qué funciones y parches se están programando en Sketchware para la versión actual y qué optimizaciones recibirá la versión Beta.
* **Laboratorio de Datos Descartados:** Documentación técnica sobre ideas de virus, scripts de prueba o lógicas de bloques que fueron descartadas durante las fases de testeo por ser inestables o provocar errores en el sistema de pruebas.
* **Control de Entornos:** Directrices técnicas para asegurar que cualquier interacción con los proyectos derivados se realice bajo estrictas normas de seguridad digital.
